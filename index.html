Verdict & Critical Fixes

Based on 25 pages · focused on routing speed and page loading

🏆 Overall Winner: RoomPoints.com

Average performance 79 vs 35. 8 of 12 pages score 80+. All 13 FlightPoints pages score under 60.

✈ FlightPoints — What's Broken

🔴Route pages are catastrophically slow. TBT on Delta route: 10,550ms — the page blocks the main thread for 10 seconds. Users can't click anything for nearly a quarter minute.
🔴Every single page fails Core Web Vitals on mobile. No page passes LCP (<2.5s). Average LCP is 11.4s across all 13 pages.
🔴TTFB is consistently 2–3.5s on most pages. The server responds slowly before any rendering begins. Getting-started page: 2,850ms TTFB.
🔴Routes listing CLS = 0.28 — well above the 0.1 threshold. Content jumps significantly as flight data loads in.
🟡JS bundles are huge and untree-shaken. Every page loads ~400 KB of unused JS. Route pages render 1,481 DOM nodes per page.

🏨 RoomPoints — What's Broken

🔴Homepage Best Practices = 0. A critical error was detected — Lighthouse gave a 0 score. This needs immediate investigation as it signals a severe code or security issue.
🔴Hotels listing LCP = 17.9s. The main hotel browsing page — the most important page for users — has the worst LCP on the entire site, driven by uncached hotel images.
🔴Wyndham TTFB = 4,850ms. One specific hotel chain page is almost 5× slower than others. Likely a cold-start or uncached backend data call for that chain.
🟡Hyatt page Speed Index = 21.5s despite low TBT. Images load slowly in the background long after the page is interactive.
🟢Strengths: IHG, Hilton, Choice, Radisson pages all score 93–98. Architecture is clean — 68 DOM nodes, ~38 requests, lean JS. The foundation is fast when data is cached.
